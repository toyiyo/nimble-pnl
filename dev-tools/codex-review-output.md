::finding:: severity=major file=src/components/scheduling/ShiftPlanner/ShiftPlannerTab.tsx line=701
The grid is replaced by the “No shift templates yet” empty state whenever `displayTemplates.length === 0`, which also happens when every template is hidden and `showHidden` is false. That bypasses `TemplateGrid`, so the `hiddenLaneByDay` passed at lines 744-745 is never rendered; a manager who hides the last active template with assigned shifts sees an empty planner instead of the required “From hidden templates” lane showing the kept schedules. Trigger: restaurant/week has only inactive templates with shifts, `showHidden` default false.
