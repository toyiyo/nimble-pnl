The split-rule branch of `apply_rules_to_bank_transactions_internal` updates `bank_transactions` without assigning `supplier_id`, while the non-split branch does so at line 713. A bank rule with `is_split_rule=true`, a description/amount criterion, and a supplier will now match supplier-less bank-feed transactions, but when the background/backfill batch engine applies it, the transaction is marked split/categorized and the rule count is incremented while `supplier_id` remains NULL. This violates the new assign-not-filter semantics specifically for split bank rules and leaves those transactions untagged by supplier.
