::finding:: severity=major file=supabase/migrations/20260705120000_cron_env_guard.sql line=76
The production marker is seeded from `EXISTS (SELECT 1 FROM public.restaurants)`, which is not production-only for any already-created preview branch or local stack that has been used for QA before this migration is applied. Trigger: create a restaurant in a preview branch, then rebase/apply this migration; line 76 inserts `('environment','production')`, `is_production()` becomes true, and the rewrapped cron jobs call `cron_invoke_edge`, which posts to the hardcoded production edge URL. That reintroduces the exact preview-to-prod cron spam this migration is meant to prevent.
