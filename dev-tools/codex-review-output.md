::finding:: severity=major file=supabase/functions/_shared/tools-registry.ts line=883
`canUseTool()` treats `get_kpis` as a basic tool for every role, and that tool returns revenue, COGS, labor, prime cost, margins, and profitability. Because this change explicitly grants `collaborator_operations_manager` `view:ai_assistant`, an external ops collaborator can open the AI assistant and ask for KPIs/profitability, bypassing the route decision that keeps them off `/` because the dashboard shows P&L. Trigger: Pro-plan restaurant, user role `collaborator_operations_manager`, AI prompt like “show this month’s profitability/KPIs”; the dispatcher allows `get_kpis` and returns P&L-like data despite the role being excluded from owner books/accounting.
